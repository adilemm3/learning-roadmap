# Complete Topic — Завершение темы

Полный цикл финализации темы: итоговая проверка → оценка → генерация всех файлов → обновление мета-файлов → верификация.

> Полная спецификация критериев: `docs/designs/2026-03-06-file-lifecycle-design.md`

## Триггеры
- Автоматически из `/learn` когда все блоки `[x]`
- Вручную: `заверши тему [название]`

## Способ выполнения

**Шаги 1-2 (итоговая проверка)** — в основном потоке, интерактивно с учеником.
**Шаги 3-6 (генерация файлов, мета-файлы)** — через фонового агента. Ученик видит только итоговое сообщение.

## Процесс

### 1. Итоговая проверка

5 вопросов возрастающей сложности по всем блокам темы:

- Вопросы покрывают разные блоки из `learning-plan.md`
- Включают: теорию, код, trade-off
- При ❌ — наводящие вопросы (не давать ответ)
- Записать результаты: ✅/❌ на каждый вопрос

### 2. Оценка уровня

По результатам калибровки + блоков + итоговой проверки определить уровень:
- **Junior** — понимает базу, путается в деталях
- **Middle** — уверенно объясняет, решает типовые задачи
- **Senior** — видит trade-off, знает нюансы, объясняет «почему»
- **Architect** — проектирует решения, видит системные связи

### 3. Генерация файлов подтемы

Сгенерировать 8 файлов. Каждый файл имеет свои источники и критерии:

| Файл | Что генерировать | Источники | Критерии |
|------|-----------------|-----------|----------|
| `cheatsheet.md` | Краткая ёмкая выжимка: суть, код, антипаттерны, таблицы | `summary.md` (сжать), `learning-plan.md` (блоки), `session.md` (инсайты) | Покрывает все блоки, есть код + антипаттерны, не копия summary |
| `interview-questions.md` | Только вопросы по 4 уровням (без ответов) | `summary.md` (концепции), `session.md` (инсайты «Собес:», ошибки) | 4 уровня (Junior/Middle/Senior/Architect), min 2 вопроса/уровень, Architect = trade-off. Ссылка на `interview-answers.md` |
| `interview-answers.md` | Вопросы + ответы в `<details>` | `interview-questions.md` (вопросы), `summary.md`, `session.md` | Каждый вопрос с ответом в `<details><summary>Ответ</summary>` |
| `practice.md` | Только задания (без решений) | `learning-plan.md` (блоки), `session.md` (слабые места), `summary.md` (концепции) | 5 задач (2 Easy + 2 Medium + 1 Hard), НЕ дублируют session.md, разные блоки и типы. Ссылка на `practice-solutions.md` |
| `practice-solutions.md` | Подсказки + решения в `<details>` с пояснениями кода | `practice.md` (задания), `summary.md`, `session.md` | Каждая задача: `<details>` подсказка + `<details>` решение. Код с комментариями к каждой строке: что делает и почему. Общая концепция перед кодом |
| `cases.md` | 2-4 production-кейса | `summary.md`, `session.md` (сценарии), знания о типичных проблемах | Min 2, структура: ситуация → причина → решение → урок, min 1 trade-off |
| `diagram.md` | ASCII/mermaid диаграммы | `summary.md` (концепции), `session.md` (схемы из объяснений) | Min 1, покрывает главные концепции |
| `resources.md` | Ссылки: docs, статьи, пакеты | Знания Claude, упомянутые в сессии ссылки | Min 1 Microsoft Docs + 1 статья + NuGet если применимо |

### 4. Финализация learning-plan.md и summary.md

- `learning-plan.md` — статус → `✅ завершена`, дата последней сессии
- `summary.md` — добавить итоговую оценку уровня

### 5. Обновление мета-файлов

**progress.md (трека):**
- Статус темы → ✅
- Уверенность (1-5) на основе итоговой проверки
- Дата последней сессии
- **Пересчитать сводную таблицу** (количество ✅ по уровням)

**Родительские файлы раздела:**
- `progress.md` раздела — статус → ✅, уверенность, дата
- `cheatsheet.md` раздела — добавить секцию из `cheatsheet.md` подтемы
- `overview.md` раздела — обновить статус подтемы

**weak-spots.md:**
- Записать пробелы из итоговой проверки (❌)
- Закрыть пробелы которые были исправлены

**learner-profile.md:**
- Обновить паттерны обучения (если накопились данные)

### 6. Верификация и предложение

- Вызвать `/verify` для проверки консистентности
- Проверить: все ли подтемы раздела завершены?
  - **Да** → предложить экзамен по разделу
  - **Нет** → предложить следующую подтему по `dependency-graph.md`

### 7. Итоговое сообщение

```
Тема «[название]» завершена! Уровень: [оценка], уверенность: [N/5].
Следующая тема: [название] или экзамен по разделу [название].
```
