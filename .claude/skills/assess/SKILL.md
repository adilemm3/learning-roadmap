# Assess — Оценка уровня

Определяет текущий уровень знаний по теме через серию вопросов.

## Триггеры
- `оцени меня`
- `оцени меня [тема]`

## Процесс

### 1. Определить тему

- Если тема указана → использовать её
- Если нет → спросить по какой теме или предложить общую оценку по нескольким темам

### 2. Задать вопросы

3-5 вопросов **возрастающей сложности**:

1. 🟢 Junior-уровень — базовые факты, определения
2. 🟡 Middle-уровень — применение, сравнение, выбор подхода
3. 🔴 Senior-уровень — внутреннее устройство, edge cases, производительность
4. 🏛 Architect-уровень — trade-offs, системный дизайн, влияние на архитектуру
5. (опционально) Практическая задача — код или код-ревью

Задавать по одному вопросу. Ждать ответа перед следующим.

Если ответ неверный на определённом уровне → можно остановиться (уровень определён).

### 3. Определить уровень

На основе ответов:
- Все Junior ✅ + часть Middle ✅ → **Junior (продвинутый)**
- Все Middle ✅ → **Middle**
- Все Senior ✅ → **Senior**
- Все Architect ✅ → **Architect**

### 4. Обновить файлы

#### progress.md
- Обновить статус темы
- Обновить уверенность (1-5):
  - 1 = не знаю тему
  - 2 = слышал, но не понимаю
  - 3 = понимаю базово
  - 4 = уверенно владею
  - 5 = могу объяснить и научить других
- Обновить дату

#### weak-spots.md
- Добавить конкретные пробелы из неверных ответов:
  `- [ ] [Тема] не знает X, путает Y с Z (YYYY-MM-DD)`

#### learner-profile.md
- Обновить если появились новые данные о стиле обучения

### 5. Результат

Вывести:
```
## Результат оценки: [тема]

Уровень: 🟡 Middle
Уверенность: 3/5

### Сильные стороны
- ...

### Пробелы
- ...

### Рекомендация
Для перехода на Senior нужно изучить: [список тем/блоков]
```

### 6. Предложить план

- «Хочешь начать изучение [рекомендованная тема]?» → вызвать `/learn`
