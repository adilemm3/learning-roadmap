# Rebalance — Балансировка дерева тем

Разбивает слишком большую тему на подтемы, создаёт иерархию родитель-дети.

## Триггеры
- `разбей тему [название]`
- Автоматически из `/learn` когда learning-plan содержит >7 блоков
- Автоматически из `/create-topic` когда тема явно составная

## Процесс

### 1. Анализ темы

Определить:
- Какие подтемы можно выделить
- Какие зависимости между подтемами
- Какой порядок изучения

### 2. Предложить разбиение

Формат предложения:
```
«[Тема] слишком широкая. Предлагаю разбить на:
1. [подтема 1] — краткое описание
2. [подтема 2] — краткое описание
...
Порядок изучения: 1 → 2 → 3
Согласен?»
```

Ждать подтверждения. Ученик может:
- Согласиться
- Предложить другое разбиение
- Отказаться (оставить как есть)

### 3. Создать структуру

#### Превратить тему в родительский узел:
1. Создать родительские файлы из `templates/section/`:
   - `overview.md` — обзор раздела
   - `cheatsheet.md` — сводная шпаргалка (пока пустая)
   - `progress.md` — таблица подтем

2. Если у темы уже были файлы (summary.md, session.md и т.д.):
   - Определить к какой подтеме относится существующий контент
   - Перенести контент в подходящую подтему
   - НЕ удалять контент — только перемещать

#### Создать подтемы:
- Для каждой подтемы вызвать логику из `/create-topic` (9 файлов из шаблона)

### 4. Обновить мета-файлы

- `roadmap.md` — заменить одну тему на группу подтем
- `progress.md` трека — заменить одну строку на несколько
- `dependency-graph.md` — заменить один узел на группу связанных узлов

### 5. Подтверждение

Вывести:
- Дерево созданных директорий
- Сводку: N подтем, порядок изучения
- Предложить начать изучение первой подтемы
