# Save Session — Сохранение прогресса

Сохраняет состояние текущей учебной сессии. Все шаги обязательны.

## Триггеры
- `сохрани` — сохранить и продолжить
- `стоп` — сохранить и завершить сессию
- Автоматически при завершении темы

## Способ выполнения

**ВСЕ файловые операции (шаги 1-6) выполнять через фонового агента** (`Agent` с `run_in_background: true`). Ученик не должен видеть diff-ы, чтения файлов и прочую техническую работу.

В основном чате — только итоговое сообщение: «Прогресс сохранён: [тема], блок [N/M]».

Передай агенту полный контекст: какие блоки пройдены, результаты проверок, фидбек, пробелы — всё что нужно для заполнения файлов.

## Чеклист сохранения

Выполни ВСЕ шаги по порядку:

### 1. session.md
Дописать диалог текущей сессии. Формат:

```markdown
## Сессия N (YYYY-MM-DD)

### Блок N: [название]

**Claude:** [объяснение]
**[YOUR_NAME]:** [вопрос/ответ]
**Claude:** [ответ]

### Проверка Блока N
**Claude:** Вопрос: ...
**[YOUR_NAME]:** ...
**Claude:** ✅ / ❌ + объяснение

**Фидбек:** понятно ✅ / частично ⚠️ / нет ❌
```

Углубления записывать как:
```markdown
#### ↳ Углубление: [тема]
**Claude:** ...
**[YOUR_NAME]:** ...
```

### 2. learning-plan.md
- Отметить пройденные блоки `[x]`
- Обновить строку `> Статус:` (⬜ → 🟡 или 🟡 → ✅)
- Обновить строку `> Блок: N/M`
- Обновить `> Последняя сессия: YYYY-MM-DD`

### 3. summary.md
- Добавить конспект новых блоков (ключевые концепции, код, нюансы)
- НЕ дублировать весь диалог — только выжимку

### 3.1. При завершении темы (статус 🟡 → ✅)

**НЕ выполнять здесь.** Вся логика завершения темы вынесена в скилл `/complete-topic`:
- Итоговая проверка (5 вопросов)
- Оценка уровня
- Генерация 6 файлов (cheatsheet, interview-questions, practice, cases, diagram, resources)
- Обновление мета-файлов и родительских файлов раздела
- Верификация

`/save-session` вызывается только для **промежуточного сохранения** (блок завершён, ученик сказал «сохрани»/«стоп»).

### 4. progress.md (трека)
- Обновить статус темы: ⬜ → 🟡 (в процессе) или 🟡 → ✅ (завершена)
- Обновить уверенность (1-5) если была проверка
- Обновить дату последней сессии
- **Пересчитать сводную таблицу** (секция «Сводка» в конце файла): посчитать количество ✅ тем по каждому уровню (Junior/Middle/Senior/Architect), обновить колонки «Изучено» и «%»

### 4.1. Родительские файлы раздела

Каждый раздел (01-csharp-fundamentals и т.д.) содержит собственные `progress.md`, `cheatsheet.md`, `overview.md`.

**progress.md раздела** — синхронизировать с progress.md трека:
- Обновить статус подтемы: ⬜ → 🟡 или 🟡 → ✅
- Обновить уверенность и дату

**cheatsheet.md раздела** — агрегация из подтем:
- Если подтема завершена (✅) → скопировать **полное содержимое** `cheatsheet.md` подтемы в раздел
- Формат: `## [Название подтемы]` + полный текст cheatsheet.md подтемы (с кодом, таблицами, антипаттернами)
- НЕ сжимать и НЕ переписывать — копировать как есть, чтобы шпаргалка была полезной
- При завершении каждой подтемы — добавлять новую секцию (не пересобирать весь файл)

**Критерии качества cheatsheet.md подтемы** (заполняется при завершении темы, шаг 3.1):
- Таблицы для сравнений и выбора (когда что использовать)
- Примеры кода для каждого ключевого паттерна
- Секция «Антипаттерны» — что НЕ делать и почему
- Параметры методов с объяснением
- Краткая и ёмкая — без воды, но без потери важного. Длина определяется объёмом темы

**overview.md раздела** — обновлять при изменении прогресса подтем:
- Таблица подтем должна включать статус (⬜/🟡/✅) и дату последней сессии
- Обновлять статус строки при каждом изменении прогресса подтемы

### 5. glossary.md (трека)

Добавить новые термины из пройденных блоков в `tracks/<трек>/glossary.md`.

**Правила:**
- Только термины, впервые встреченные в этом блоке
- Вставлять в алфавитном порядке (по английскому названию)
- Если термин уже есть — не дублировать
- Формат:
```markdown
**Термин** — определение (1-2 предложения).
Тема: [подтема]/блок [N] | См. также: [связанные термины]
```

### 6. weak-spots.md

**Что считается пробелом:**
- ❌ неверный ответ на проверочный вопрос
- Ученик сказал «не понимаю» или фидбек = «нет ❌»
- Углубление выявило незнание связанной темы
- Ученик не смог решить практическую задачу

**Формат записи:**
```markdown
## Активные
- [ ] [Type System] не понимает checked/unchecked arithmetic (2026-03-05)
- [ ] [Async] путает Task.Run и async/await — не знает когда какой (2026-03-05)
```

**Закрытие пробела:**
- Пробел считается закрытым когда ученик правильно ответил на вопрос по этой теме в последующей сессии
- Перенести в «Закрытые» с датой: `- [x] [Type System] checked/unchecked (закрыто: 2026-03-10)`

**НЕ записывать как пробел:**
- Тему которую ещё не начинали изучать
- Вопрос Architect-уровня если ученик на Junior

### 7. learner-profile.md

**Когда обновлять:**

#### Секция «Стиль обучения»
Обновлять после 2-3 сессий когда виден паттерн:
- `Темп` — если ученик часто говорит «пропусти» → «быстрый». Если часто «углубись» → «детальный»
- `Глубина` — если часто задаёт вопросы «а почему?» → «любит углубляться»
- `Практика` — если просит задачи до теории → «сначала задача»

#### Секция «Что работает»
Записывать конкретные наблюдения:
```markdown
## Что работает
- Аналогии с реальным миром для абстрактных тем (сессия type-system 2026-03-05)
- Короткие блоки по 5-7 минут (сессии gc, async)
- Код-примеры сразу после теории
```

#### Секция «Что НЕ работает»
```markdown
## Что НЕ работает
- Длинные теоретические объяснения без кода — теряет фокус (сессия clr-jit 2026-03-12)
- Слишком простые вопросы — скучает (сессия arrays-lists)
```

#### Секция «Паттерны обучения»
Заполняется когда из микрофидбека видны устойчивые паттерны (3+ сессий):
```markdown
## Паттерны обучения
- Лучше усваивает утром (высокие оценки в утренних сессиях)
- Предпочитает визуальные схемы для архитектурных тем
- Сильнее в практике чем в теории — быстро решает задачи, но путается в определениях
```

**НЕ записывать:**
- Наблюдения после одной сессии (кроме явных предпочтений)
- Общие фразы без конкретики («хорошо учится»)

### 8. Верификация

После шагов 1-7 — вызвать скилл `/verify` для проверки консистентности.

### 9. Подтверждение
- Вывести: «Прогресс сохранён: [тема], блок [N/M]»
- Вывести краткий результат верификации
- Если команда «стоп» → спросить: «Запушить?»
- Если команда «сохрани» → продолжить обучение
