# Create Topic — Создание новой темы

Создаёт новую тему с полной структурой файлов и обновляет все мета-файлы.

## Триггеры
- `добавь тему [название]`
- `новая тема [название]`
- Автоматически из скилла `/learn` когда тема не найдена

## UX-правило

**После подтверждения пользователя — МОЛЧА выполни шаги 3-4 и покажи только итог.**
Не выводи промежуточные статусы («читаю файлы», «создаю тему», «обновляю мета-файлы»).
Шаги 3-4 выполняй через фоновых агентов. Пользователь видит только шаг 1-2 (до подтверждения) и шаг 5 (итог).

## Процесс

### 1. Определить место в дереве

Спросить или определить:
- **Трек** — в каком треке (dotnet, golang и т.д.)
- **Раздел** — существующий или новый (01-csharp-fundamentals, 07-data-access и т.д.)
- **Уровень** — Junior / Middle / Senior / Architect
- **Зависимости** — какие темы нужно изучить перед этой

### 2. Проверить размер темы

Если тема явно составная (SQL, базы данных, брокеры сообщений):
- Предложить разбиение на подтемы → вызвать `/rebalance`
- Ждать подтверждения

### 3. Создать директорию и файлы

Запустить скрипт (создаёт 9 файлов из шаблонов + git add):
```bash
./scripts/create-topic.sh "<section>" "<topic-dir>" "<Topic Name>" "<Level>" "<dependencies>"
```
Пример:
```bash
./scripts/create-topic.sh "06-asp-net-core" "08-api-clients-http" "API Clients & HTTP" "Middle" "Dependency Injection, Middleware Pipeline"
```

### 4. Обновить мета-файлы (6 точечных правок)

Каждое обновление — добавить 1 строку, без перенумерации:
- `tracks/dotnet/progress.md` — добавить строку в нужную секцию (статус ⬜, уверенность 0)
- `tracks/dotnet/roadmap.md` — добавить тему в соответствующий уровень с зависимостями
- `tracks/dotnet/dependency-graph.md` — добавить узел и связи в mermaid-граф
- `INDEX.md` — добавить строку в таблицу секции
- `tracks/dotnet/topics/<section>/overview.md` — добавить строку в таблицу подтем **с зависимостями** и обновить секцию «Порядок изучения и зависимости»:
  - Добавить строку в таблицу: `| Новая тема | Зависит от: [список тем] |`
  - Добавить шаг в нумерованный порядок изучения в правильное место по зависимостям
  - Ученик должен видеть что нужно изучить сначала перед переходом к новой теме
- `tracks/dotnet/topics/<section>/progress.md` — добавить строку

### File Lifecycle

> Полная спецификация: `docs/designs/2026-03-06-file-lifecycle-design.md`

**Чеклист `/create-topic` — начальное состояние файлов:**

Скрипт `create-topic.sh` создаёт 9 файлов из шаблонов. Все файлы на этом этапе — пустые шаблоны с `{{TOPIC_NAME}}` заменённым. Контент появляется позже (при `/learn` и `/save-session`).

| Файл | Начальное состояние |
|------|-------------------|
| `learning-plan.md` | Шаблон: статус ⬜, калибровка пуста, блоков нет |
| `session.md` | Заголовок + HTML-комментарий |
| `summary.md` | Шаблон с пустыми секциями |
| `cheatsheet.md` | Шаблон с пустыми секциями |
| `interview-questions.md` | Шаблон: 4 уровня (Junior/Middle/Senior/Architect), вопросов нет |
| `practice.md` | Шаблон: 5 задач (2 Easy + 2 Medium + 1 Hard), условий нет |
| `cases.md` | Шаблон с пустыми секциями |
| `diagram.md` | Шаблон с пустыми секциями |
| `resources.md` | Шаблон с пустыми секциями |

Родительские файлы раздела (если раздел новый) создаются из `templates/section/`.

### 5. Итог (единственное сообщение после подтверждения)

Краткий итог в 1-2 строки:
```
Тема «[название]» создана: `[путь]`. Начать изучение? → `учим`
```
