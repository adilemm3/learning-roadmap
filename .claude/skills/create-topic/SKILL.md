# Create Topic — Создание новой темы

Создаёт новую тему с полной структурой файлов и обновляет все мета-файлы.

## Триггеры
- `добавь тему [название]`
- `новая тема [название]`
- Автоматически из скилла `/learn` когда тема не найдена

## Процесс

### 1. Определить место в дереве

Спросить или определить:
- **Трек** — в каком треке (dotnet, golang и т.д.)
- **Раздел** — существующий или новый (01-csharp-fundamentals, 07-data-access и т.д.)
- **Уровень** — Junior / Middle / Senior / Architect
- **Зависимости** — какие темы нужно изучить перед этой

### 2. Проверить размер темы

Если тема явно составная (SQL, базы данных, брокеры сообщений):
- Предложить разбиение на подтемы → вызвать `/rebalance`
- Ждать подтверждения

### 3. Создать директорию и файлы

Из шаблона `templates/topic/`:
1. Скопировать все 9 файлов
2. Заменить плейсхолдеры:
   - `{{TOPIC_NAME}}` → название темы
   - `{{LEVEL}}` → уровень
   - `{{DEPENDENCIES}}` → список зависимостей

### 4. Обновить мета-файлы

- `roadmap.md` — добавить тему в соответствующий уровень с зависимостями
- `progress.md` — добавить строку (статус ⬜, уверенность 0)
- `dependency-graph.md` — добавить узел и связи в mermaid-граф

### 5. Обновить родительские файлы (если есть)

Если тема создаётся внутри существующего раздела с родительскими файлами:
- Обновить `overview.md` родителя
- Обновить `progress.md` родителя

### 6. Git

- `git add` новых файлов

### 7. Подтверждение

Вывести:
- Путь к созданной теме
- Список файлов
- Место в roadmap
- Предложить: «Начать изучение?» → вызвать `/learn`
