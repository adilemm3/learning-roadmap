# Verify — Валидация и верификация

Проверяет что все файлы обновлены корректно и консистентны между собой. Запускается автоматически после `/save-session` или по запросу.

> Полная спецификация критериев: `docs/designs/2026-03-06-file-lifecycle-design.md`

## Триггеры
- `проверь` / `verify` — ручной запуск
- Автоматически после каждого `/save-session`

## Чеклист проверки

### 1. session.md
- [ ] Файл существует для активной темы
- [ ] Есть запись текущей сессии с датой `## Сессия N (YYYY-MM-DD)`
- [ ] Калибровка записана (если это первая сессия темы)
- [ ] Все блоки сессии записаны (сверить с learning-plan — каждый `[x]` блок отражён)
- [ ] Для каждого блока: объяснение, проверочные вопросы, микрофидбек
- [ ] Формат соблюдён: `**Claude:**` / `**Адиль:**` / `**Фидбек:**`
- [ ] Итоговая проверка с оценкой (если тема завершена)

### 2. learning-plan.md
- [ ] Строка `> Статус:` соответствует реальности (⬜ / 🟡 / ✅)
- [ ] Строка `> Блок: N/M` — N = количество `[x]` блоков
- [ ] Строка `> Последняя сессия:` = сегодняшняя дата
- [ ] Калибровка заполнена (не пустой шаблон)
- [ ] Каждый пройденный блок отмечен `[x]`
- [ ] Каждый непройденный блок — `[ ]`

### 3. summary.md
- [ ] Для каждого `[x]` блока в learning-plan есть соответствующая секция
- [ ] Каждая секция содержит: ключевые концепции, примеры кода, нюансы
- [ ] Итоговая оценка уровня (если тема завершена)
- [ ] Не является дословной копией session.md — переработанный конспект

### 4. cheatsheet.md (подтемы) — только для завершённых тем (✅)
- [ ] Не пустой (не шаблон)
- [ ] Есть секция с кодом
- [ ] Есть секция антипаттернов или подводных камней
- [ ] Покрывает все блоки из learning-plan.md
- [ ] Не дублирует summary.md дословно — сжатая выжимка

### 5. interview-questions.md — только для завершённых тем (✅)
- [ ] 4 уровня: Junior, Middle, Senior, Architect
- [ ] Минимум 2 вопроса на уровень
- [ ] Только вопросы, без ответов
- [ ] Ссылка на `interview-answers.md`
- [ ] Вопросы покрывают разные блоки темы
- [ ] Architect-вопросы про trade-off и архитектурные решения

### 5.1. interview-answers.md — только для завершённых тем (✅)
- [ ] Каждый вопрос из `interview-questions.md` присутствует
- [ ] Каждый ответ в `<details><summary>Ответ</summary>`
- [ ] Нумерация совпадает с `interview-questions.md`

### 6. practice.md — только для завершённых тем (✅)
- [ ] Ровно 5 задач
- [ ] Градация: 2 Easy + 2 Medium + 1 Hard
- [ ] Только задания, без подсказок и решений
- [ ] Ссылка на `practice-solutions.md`
- [ ] Задачи НЕ дублируют задачи из session.md
- [ ] Задачи покрывают разные блоки из learning-plan.md
- [ ] Типы задач разнообразны (не все одного типа)

### 6.1. practice-solutions.md — только для завершённых тем (✅)
- [ ] Каждая задача из `practice.md` присутствует
- [ ] Каждая подсказка в `<details><summary>Подсказка</summary>`
- [ ] Каждое решение в `<details><summary>Решение</summary>`
- [ ] Код с комментариями: что делает каждая строка и почему
- [ ] Общая концепция перед кодом, ключевые решения после

### 7. cases.md — только для завершённых тем (✅)
- [ ] Минимум 2 кейса
- [ ] Каждый кейс: ситуация + причина + решение + урок
- [ ] Кейсы разнообразны (не все про одну проблему)
- [ ] Минимум 1 кейс — trade-off или постмортем

### 8. diagram.md — только для завершённых тем (✅)
- [ ] Минимум 1 диаграмма
- [ ] Покрывает главные концепции темы
- [ ] Формат: ASCII или mermaid (читается без рендера)

### 9. resources.md — только для завершённых тем (✅)
- [ ] Минимум 1 ссылка на официальную документацию (Microsoft Docs)
- [ ] Минимум 1 статья/блог
- [ ] NuGet пакеты перечислены (если тема их использует)

### 10. progress.md (трека)
- [ ] Строка активной темы имеет статус 🟡 или ✅
- [ ] Дата последней сессии = сегодня
- [ ] Уверенность обновлена (если была проверка)
- [ ] Сводная таблица пересчитана: количество ✅ по каждому уровню = реальное число завершённых тем этого уровня

### 11. weak-spots.md
- [ ] Каждый ❌ из проверки блоков записан как пробел
- [ ] Формат: `- [ ] [Тема] описание пробела (YYYY-MM-DD)`
- [ ] Нет дубликатов с уже существующими пробелами
- [ ] Если пробел закрыт в этой сессии — перенесён в «Закрытые»

### 12. learner-profile.md
- [ ] Если были новые наблюдения о стиле — записаны
- [ ] Секция «Что работает» / «Что НЕ работает» обновлена (если был фидбек)
- [ ] Секция «Паттерны обучения» обновлена (если накопились данные)

### 13. Родительские файлы раздела
- [ ] `progress.md` раздела — статус подтемы соответствует learning-plan
- [ ] `cheatsheet.md` раздела — секция для каждой завершённой (✅) подтемы
- [ ] `overview.md` раздела — статусы подтем актуальны

### 14. Консистентность между файлами
- [ ] Количество `[x]` в learning-plan = количество секций в summary
- [ ] Статус в progress.md трека = статус в learning-plan
- [ ] Статус в progress.md раздела = статус в progress.md трека
- [ ] Пробелы из session.md (❌) присутствуют в weak-spots.md
- [ ] Если тема завершена (✅) — все блоки `[x]`, есть итоговая проверка, все 11 файлов заполнены
- [ ] `glossary.md` трека содержит термины из всех пройденных блоков

## Формат вывода

```
## Верификация: [тема]

✅ session.md — записано
✅ learning-plan.md — блок 3/5, статус 🟡
✅ summary.md — конспект блоков 1-3
✅ progress.md — обновлено
⚠️ weak-spots.md — пробел "checked arithmetic" не записан
✅ learner-profile.md — без изменений
✅ Консистентность — ок

### Исправления
- Добавлен пробел в weak-spots.md: checked arithmetic
```

## При обнаружении проблем

1. **Автоисправление** — если проблема однозначная (не записан пробел, не обновлена дата) → исправить автоматически
2. **Запрос** — если проблема неоднозначная (конфликт данных, пропущен блок) → спросить у ученика
3. **Лог** — всегда выводить результат проверки, даже если всё ок
