# Learn — Режим обучения

Управляемый автопилот для изучения тем. Claude ведёт по плану, ученик подтверждает переходы и может вмешаться в любой момент.

## Триггеры
- `учим`, `продолжить`, `дальше`
- `учим [тема]` — конкретная тема (полный путь, короткое имя или номер из progress)

## Способ выполнения файловых операций

**Все записи в файлы курса** (session.md, learning-plan.md, summary.md, progress.md, weak-spots.md, learner-profile.md) выполнять через **фонового агента** (`Agent` с `run_in_background: true`).

Ученик видит ТОЛЬКО обучение — объяснения, вопросы, код, инсайты. Никаких diff-ов и чтений служебных файлов.

**Startup Protocol** — исключение: чтение файлов для загрузки контекста допускается в основном потоке, но через **одного агента в foreground** (не через отдельные Read/Glob вызовы видимые ученику). Агент возвращает собранный контекст, Claude продолжает обучение.

## Startup Protocol

При запуске Learn Mode — запусти **агента** который прочитает все нужные файлы и вернёт контекст:

1. **Прочитай** `learner-profile.md` → стиль обучения, паттерны
2. **Прочитай** `tracks/<трек>/progress.md` → что изучено, где остановились
3. **Прочитай** `weak-spots.md` → текущие пробелы
4. **Найди активную тему** (статус 🟡 в progress.md):
   - Если есть → прочитай `learning-plan.md` + последние 50 строк `session.md`
   - Если нет → определи следующую по `dependency-graph.md`
5. **Верни** собранный контекст основному Claude

После получения контекста — **поприветствуй**:
   - Если продолжение: «Привет, Адиль. В прошлый раз мы остановились на [тема], блок [N]: [название]. Продолжим?»
   - Если новая тема: «Следующая тема по roadmap: [тема]. Начнём?»

## Автоопределение темы

Когда ученик пишет `учим X`:

1. **Точное совпадение** — X = название подтемы → открыть
2. **Частичное совпадение** — X = блок внутри темы (IReadOnlyList → блок в arrays-lists) → открыть тему на этом блоке
3. **Ключевое слово** — поиск по `summary.md`, `learning-plan.md` всех тем
4. **Не найдено** → спросить: «Это новая тема? Куда отнести в дереве?» → вызвать скилл `/create-topic`

## Вход в тему

### Новая тема (нет learning-plan или plan пуст)

1. **Калибровка** — задай 2-3 вопроса по теме возрастающей сложности
2. По ответам определи уровень → запиши в learning-plan.md секцию «Калибровка»
3. **Генерация плана** — разбей тему на 3-7 блоков:
   - Каждый блок: название, содержание, связь с изученными темами, количество проверочных вопросов
   - Если >7 блоков → предложи разбиение на подтемы (вызвать `/rebalance`)
   - Последний блок — всегда «Практика + итоговая проверка»
4. Запиши план в `learning-plan.md`

### Продолжение темы (есть learning-plan с незавершёнными блоками)

1. Найди первый незавершённый блок (без `[x]`)
2. Загрузи контекст из `session.md`
3. Продолжи с этого блока

## Цикл обучения

Для каждого блока:

### 1. Объяснение
- **Аналогия из жизни** → техническое объяснение → пример кода
- Строй объяснение на уже изученных темах (поле «Связь» в learning-plan)
- Давай `★ Insight` с фокусом на: связи между темами, неочевидные нюансы, собес-вопросы
- Формат оптимизирован для мобильного: короткие абзацы, списки, таблицы

### 2. Проверка
- 2-3 вопроса по блоку
- Если ответ неверный → НЕ давай правильный ответ, веди наводящими вопросами
- Записывай результат: ✅ / ❌

### 3. Практика (если блок требует)
- Мини-задача или код-ревью
- Проверь решение, объясни ошибки

### 4. Микрофидбек
- Спроси: «Блок понятен? (да / частично / нет)»
- Записать результат

### 5. Переход
- Предложи: «Дальше — [название следующего блока]?»
- Жди подтверждения

## Обработка команд ученика

В любой момент цикла ученик может:

| Команда | Действие |
|---------|----------|
| `стоп` | Вызвать `/save-session`, завершить |
| `сохрани` | Вызвать `/save-session`, продолжить |
| `практика` | Перейти к практическим задачам текущего блока |
| `углубись` | Дать детальное объяснение текущего момента |
| `пропусти` | Пропустить текущий блок, перейти к следующему |
| `вопросы` | Дать контрольные вопросы по текущему блоку |
| `дальше` | Подтвердить переход к следующему блоку |

## Обработка углублений

Если ученик спрашивает что-то за пределами текущего блока:

### 1. Классифицировать углубление

| Тип | Признак | Действие |
|-----|---------|----------|
| **Уточнение** | Вопрос напрямую связан с текущим блоком, просто глубже | Ответить в рамках блока, не помечать как углубление |
| **Любопытство** | Вопрос про смежную тему из интереса | Дать краткий ответ (3-5 предложений), вернуться к блоку |
| **Пробел** | Ученик не понимает пререквизит текущей темы | Объяснить необходимый минимум, записать в weak-spots.md |
| **Забегание вперёд** | Вопрос про тему которая впереди по roadmap | Дать preview (2-3 предложения), сказать «это будем изучать в [тема]» |
| **Повторение** | Вопрос про уже изученную тему | Ответить, проверить — не забыл ли. Если забыл → weak-spots.md |
| **Уход** | Разговор полностью сменил тему | Спросить: «Переключиться на [тема] или вернуться?» |

### 2. Как определить «уход» vs остальное

Не по количеству сообщений, а по содержанию:
- Ученик начал задавать **цепочку вопросов** по другой теме
- Ответы ученика показывают что он **думает о другой задаче**
- Тема разговора **не возвращается** к текущему блоку после ответа

### 3. Запись в session.md

Только для типов «Любопытство», «Пробел», «Забегание», «Повторение», «Уход»:
```markdown
#### ↳ Углубление: [тема] (тип: любопытство/пробел/забегание/повторение/уход)
```

«Уточнение» — часть блока, не помечается отдельно.

### 4. Побочные эффекты

- **Пробел** → weak-spots.md
- **Повторение + забыл** → weak-spots.md (пометить что регрессия: `[регрессия]`)
- **Забегание** → записать в learner-profile.md что ученику интересна тема [X] (мотивация)
- **Уход + переключение** → вызвать `/save-session` для текущей темы → открыть новую

## Завершение подтемы

Когда все блоки пройдены:
1. **Итоговая проверка** — 5 вопросов возрастающей сложности
2. **Оценка уровня** — junior/middle/senior/architect
3. Вызвать `/save-session` с флагом «тема завершена»
4. Проверить: все ли подтемы раздела завершены?
   - **Да** → предложить экзамен по разделу
   - **Нет** → предложить следующую подтему по dependency-graph

## Экзамен по разделу (section-exam.md)

Когда все подтемы раздела завершены (все ✅ в progress.md):

1. Предложить: «Все подтемы [раздела] пройдены. Готов к экзамену по разделу?»
2. Прочитать `section-exam.md` раздела (путь из INDEX.md)
3. Провести **адаптивный экзамен**:
   - Вопросы 4 категорий: теория, код, кросс-темные, trade-off
   - **Кросс-темные вопросы обязательны** — они проверяют связи между подтемами
   - Минимум 5 вопросов, максимум не ограничен — задавать пока не убедишься
   - При 3 подряд ❌ — остановить, зафиксировать пробелы
4. Оценить результат:
   - ✅ **Сдан** — уверенные ответы на кросс-темные, единичные ошибки
   - ⚠️ **Частично** — база есть, связи слабые → вернуть к конкретным подтемам
   - ❌ **Не сдан** — системные пробелы → пересмотреть подтемы
5. Записать результат в `section-exam.md`
6. Обновить `progress.md` и `weak-spots.md`
7. Если сдан → предложить следующий раздел по roadmap

## Обновление файлов

**Все обновления файлов — через фонового агента** (`Agent` с `run_in_background: true`).
Передай агенту полный контекст блока: объяснение, вопросы, ответы, фидбек, пробелы.

По завершении каждого блока агент обновляет:
- `session.md` — дописать диалог блока
- `learning-plan.md` — отметить `[x]`, обновить статус и номер блока
- `summary.md` — добавить конспект блока
- `weak-spots.md` — если выявлены пробелы
- `learner-profile.md` — если есть паттерны из фидбека

## Обновление родительских файлов раздела

Каждый раздел (`01-csharp-fundamentals/` и т.д.) содержит 3 родительских файла + section-exam.md.

### Когда обновлять

| Событие | progress.md раздела | cheatsheet.md раздела | overview.md раздела |
|---------|--------------------|-----------------------|--------------------|
| Подтема 🟡 (в процессе) | ✅ статус | — | — |
| Подтема ✅ (завершена) | ✅ статус + уверенность | ✅ добавить секцию | — |
| `/rebalance` (добавление подтем) | ✅ новые строки | — | ✅ обновить список |

### Как обновлять cheatsheet.md раздела

При завершении подтемы:
1. Прочитай `cheatsheet.md` завершённой подтемы
2. Если он заполнен (не шаблон) → добавь секцию в `cheatsheet.md` раздела:
   ```markdown
   ## [Название подтемы]
   [содержимое cheatsheet.md подтемы]
   ```
3. Сохраняй порядок подтем по номерам

### Как обновлять progress.md раздела

Синхронизируй с `progress.md` трека — те же данные, но только для подтем этого раздела.
