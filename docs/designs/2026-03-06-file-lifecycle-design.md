# File Lifecycle — Жизненный цикл файлов

> Дата: 2026-03-06
> Статус: draft
> Контекст: спецификация когда и чем заполняется каждый файл подтемы и раздела

## Зачем этот документ

Единый источник правды о жизненном цикле всех файлов системы обучения.
Скиллы (`/create-topic`, `/learn`, `/save-session`) ссылаются на него для своих чеклистов.
Скилл `/verify` использует критерии полноты для валидации.

---

## Этапы жизненного цикла

| Этап | Триггер | Скилл |
|------|---------|-------|
| **Создание темы** | `/create-topic` или `/learn` (тема не найдена) | `/create-topic` |
| **Начало изучения** | `/learn` — калибровка + генерация плана | `/learn` |
| **Конец блока** | микрофидбек «да» → переход к следующему блоку | `/learn` (фоновый агент) |
| **Конец темы** | все блоки `[x]` + итоговая проверка пройдена | `/save-session` |
| **По запросу** | `шпаргалка`, `практика`, `вопросы`, `кейс`, `схема` | ручные команды |

## Листовой узел — 9 файлов подтемы

### learning-plan.md — План изучения

**Назначение:** структура темы — блоки, прогресс, статус.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание темы | `/create-topic` | Файл из шаблона, `{{TOPIC_NAME}}` заменён |
| Начало изучения | `/learn` | Калибровка заполнена, блоки сгенерированы `- [ ]`, статус `🟡`, `Блок: 0/N` |
| Конец блока | `/learn` | Блок отмечен `[x]`, `Блок: K/N` обновлён |
| Конец темы | `/save-session` | Статус `✅ завершена`, все `[x]`, дата последней сессии |

**Источники:** генерируется Claude на основе калибровки ученика и scope темы.

**Критерии полноты (`/verify`):**
- [ ] Статус соответствует реальному (🟡 если в процессе, ✅ если завершена)
- [ ] Калибровка заполнена (не пустой шаблон)
- [ ] Все блоки перечислены с описанием
- [ ] Завершённые блоки отмечены `[x]`
- [ ] Номер текущего блока (`Блок: K/N`) актуален
- [ ] Дата последней сессии актуальна

### session.md — Сессии обучения

**Назначение:** полный диалог всех учебных сессий — калибровка, объяснения, Q&A, практика, углубления.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание темы | `/create-topic` | Файл из шаблона (пустой заголовок) |
| Начало изучения | `/learn` | Заголовок сессии + дата + калибровочный диалог |
| Конец блока | `/learn` | Дописывается: объяснение, проверка (Q&A), микрофидбек, углубления |
| Конец темы | `/learn` | Дописывается итоговая проверка с оценкой |

**Источники:** записывается в реальном времени из диалога ученик-Claude.

**Критерии полноты (`/verify`):**
- [ ] Калибровка записана
- [ ] Каждый блок из learning-plan.md отражён (заголовок + содержимое)
- [ ] Для каждого блока есть: объяснение, проверочные вопросы, микрофидбек
- [ ] Итоговая проверка с оценкой (если тема завершена)
- [ ] Даты сессий проставлены

### summary.md — Конспект

**Назначение:** структурированный конспект по всем блокам — ключевые концепции, код, нюансы.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание темы | `/create-topic` | Файл из шаблона (пустой) |
| Конец блока | `/save-session` | Добавляется конспект завершённого блока |
| Конец темы | `/complete-topic` | Финализация: все блоки + итоговая оценка |

**Источники:**
- `session.md` — объяснения, инсайты, код из диалога
- `learning-plan.md` — список блоков (каждый блок = секция в конспекте)

**Критерии полноты (`/verify`):**
- [ ] Секция на каждый блок из learning-plan.md
- [ ] Каждая секция содержит: ключевые концепции + примеры кода
- [ ] Нюансы и подводные камни отражены
- [ ] Итоговая оценка уровня (если тема завершена)
- [ ] Не является дословной копией session.md — это переработанный конспект

### cheatsheet.md — Шпаргалка

**Назначение:** краткая и ёмкая выжимка для быстрого повторения. Каждое предложение несёт информацию, без воды. Длина определяется объёмом темы, не искусственным лимитом.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание темы | `/create-topic` | Файл из шаблона (пустой) |
| Конец темы | `/complete-topic` | Сгенерировать целиком |
| По запросу | `шпаргалка [тема]` | Регенерация |

**Источники:**
- `summary.md` — основа, сжать до выжимки
- `learning-plan.md` — список блоков (каждый блок должен быть представлен)
- `session.md` — инсайты и нюансы из обсуждения

**Обязательные секции:**
- Краткая суть (2-3 предложения)
- Ключевые моменты (bullet points)
- Код (минимальные рабочие примеры)
- Антипаттерны (что не делать)

**Опциональные секции (если применимо):**
- Сравнительная таблица
- Мнемоники / правила

**Критерии полноты (`/verify`):**
- [ ] Не пустой (не шаблон)
- [ ] Есть секция с кодом
- [ ] Есть секция антипаттернов или подводных камней
- [ ] Покрывает все блоки из learning-plan.md
- [ ] Не дублирует summary.md дословно — это сжатая выжимка, не копия

### interview-questions.md — Вопросы для собеседования

**Назначение:** вопросы по уровням (Junior, Middle, Senior, Architect) с ответами.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание темы | `/create-topic` | Файл из шаблона (пустой, 4 уровня) |
| Конец темы | `/complete-topic` | Сгенерировать вопросы по всем 4 уровням |
| По запросу | `вопросы [тема]` | Дополнение/обновление |

**Источники:**
- `summary.md` — ключевые концепции для формулировки вопросов
- `session.md` — инсайты с пометкой «Собес:», ошибки ученика (типичные заблуждения)
- `learning-plan.md` — блоки определяют scope вопросов

**Критерии полноты (`/verify`):**
- [ ] 4 уровня: Junior, Middle, Senior, Architect
- [ ] Минимум 2 вопроса на уровень
- [ ] Каждый вопрос имеет ответ (inline или в `<details>`)
- [ ] Вопросы покрывают разные блоки темы
- [ ] Architect-вопросы про trade-off и архитектурные решения

### practice.md — Практические задачи

**Назначение:** 5 новых задач для самостоятельной отработки после завершения темы. Задачи из учебных сессий хранятся в session.md, здесь — только новые.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание темы | `/create-topic` | Файл из шаблона (пустой) |
| Конец темы | `/complete-topic` | Сгенерировать 5 задач |
| По запросу | `практика [тема]` | Дополнить/заменить задачи |

**Источники:**
- `learning-plan.md` — список блоков (задачи покрывают разные блоки)
- `session.md` — калибровка + ошибки ученика (задачи на слабые места)
- `summary.md` — ключевые концепции (задачи на применение)

Задачи из session.md не копируются — они служат ориентиром чтобы новые задачи были другими.

**Формат задачи:**
```
## Задача N: [название]
> Сложность: Easy/Medium/Hard | Тип: [код-ревью / реализация / рефакторинг / дебаг / архитектура]

[условие — видно сразу]

<details><summary>Подсказка</summary>
[наводящая мысль, не ответ]
</details>

<details><summary>Решение</summary>
[развёрнутый ответ с кодом и пояснением]
</details>
```

**Градация сложности:** Easy, Easy, Medium, Medium, Hard (2+2+1)

**Критерии полноты (`/verify`):**
- [ ] Ровно 5 задач
- [ ] Градация: 2 Easy + 2 Medium + 1 Hard
- [ ] Каждая задача: условие + `<details>` подсказка + `<details>` решение
- [ ] Задачи НЕ дублируют задачи из session.md
- [ ] Задачи покрывают разные блоки из learning-plan.md
- [ ] Типы задач разнообразны (не все одного типа)
- [ ] Hard-задача учитывает ошибки из калибровки/сессии (слабые места)

### cases.md — Реальные кейсы

**Назначение:** реальные production-кейсы: ситуация, причина, решение, урок. Постмортемы и trade-off анализ.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание темы | `/create-topic` | Файл из шаблона (пустой) |
| Конец темы | `/complete-topic` | Сгенерировать 2-4 кейса |
| По запросу | `кейс [тема]` | Дополнительные кейсы |

**Источники:**
- `summary.md` — ключевые концепции (кейсы иллюстрируют их применение)
- `session.md` — обсуждённые сценарии и trade-off
- Знания Claude о типичных production-проблемах по теме

**Критерии полноты (`/verify`):**
- [ ] Минимум 2 кейса
- [ ] Каждый кейс: ситуация + причина + решение + урок
- [ ] Кейсы разнообразны (не все про одну проблему)
- [ ] Хотя бы 1 кейс — trade-off или постмортем

## Родительский узел — 4 файла раздела

### overview.md — Обзор раздела

**Назначение:** описание раздела, список подтем, порядок изучения, зависимости, связи.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание раздела | `/create-topic` | Генерация целиком: описание, подтемы, порядок, связи |
| `/rebalance` | `/rebalance` | Обновить список подтем и порядок |
| Подтема завершена | `/complete-topic` | Обновить статус подтемы в таблице |

**Источники:**
- `roadmap.md` — общая структура трека
- `dependency-graph.md` — зависимости между темами

**Критерии полноты (`/verify`):**
- [ ] Все подтемы перечислены с описанием
- [ ] Порядок изучения с зависимостями
- [ ] Связи с другими разделами
- [ ] Статусы подтем актуальны (соответствуют progress.md раздела)

### progress.md — Прогресс раздела

**Назначение:** таблица прогресса по подтемам раздела.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание раздела | `/create-topic` | Таблица: все подтемы со статусом ⬜ |
| Подтема начата | `/learn` | ⬜ → 🟡, дата сессии |
| Подтема завершена | `/complete-topic` | 🟡 → ✅, уверенность 1-5, дата |
| `/rebalance` | `/rebalance` | Новые подтемы добавлены с ⬜ |

**Источники:**
- Статусы из `learning-plan.md` каждой подтемы

**Критерии полноты (`/verify`):**
- [ ] Все подтемы раздела в таблице
- [ ] Статусы соответствуют learning-plan.md каждой подтемы
- [ ] Уверенность (1-5) проставлена для каждой ✅ подтемы
- [ ] Даты последних сессий актуальны

### cheatsheet.md (раздела) — Сводная шпаргалка

**Назначение:** агрегация шпаргалок завершённых подтем.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание раздела | `/create-topic` | Файл из шаблона (пустой заголовок) |
| Подтема завершена | `/complete-topic` | Добавить секцию из cheatsheet.md подтемы |
| Все подтемы завершены | `/save-session` | Полная сводка |

**Источники:**
- `cheatsheet.md` каждой завершённой подтемы

**Критерии полноты (`/verify`):**
- [ ] Секция на каждую завершённую (✅) подтему
- [ ] Содержимое соответствует cheatsheet.md дочерних тем
- [ ] Не пустой если есть хотя бы одна ✅ подтема

### section-exam.md — Экзамен по разделу

**Назначение:** адаптивный экзамен после завершения всех подтем раздела.

**Lifecycle:**

| Этап | Скилл | Действие |
|------|-------|----------|
| Создание раздела | `/create-topic` | Файл из шаблона: формат, правила, критерии |
| Все подтемы ✅ | — | Предлагается сдача экзамена |
| После попытки | `/assess` | Запись результата: дата, вопросы, итог, пробелы |

**Источники:**
- `summary.md` + `cheatsheet.md` всех подтем — для формулировки вопросов
- `weak-spots.md` — для акцента на пробелах

**Критерии полноты (`/verify`):**
- [ ] Формат и правила заполнены (из шаблона)
- [ ] Результаты попыток — только если экзамен был сдан
- [ ] Не заполняется до завершения всех подтем раздела
