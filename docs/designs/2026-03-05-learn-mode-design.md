# Learn Mode — Автоматизированная система обучения

> Дата: 2026-03-05
> Статус: Draft
> Автор: Адиль + Claude

## Цель

Превратить learning-roadmap из набора шаблонов в **автопилотную систему обучения**, где Claude ведёт по структурированному плану, сохраняет полный диалог, адаптируется под ученика и позволяет прервать/продолжить в любой момент.

---

## 1. Концепция Learn Mode

### Управляемый автопилот

Claude ведёт сессию по плану, но ученик может вмешаться в любой момент:

| Команда | Действие |
|---------|----------|
| `учим` / `продолжить` | Продолжить активную тему или предложить следующую |
| `учим [тема]` | Начать/продолжить конкретную тему |
| `стоп` | Сохранить прогресс, завершить сессию |
| `практика` | Перейти к практическим задачам |
| `углубись` | Детальное объяснение текущего момента |
| `пропусти` | Перейти к следующему блоку |
| `вопросы` | Контрольные вопросы по текущему блоку |
| `сохрани` | Записать прогресс без завершения |
| `дальше` | Подтвердить переход к следующему блоку |

### Цикл обучения

```
┌─────────────────────────────────────────────────┐
│  "учим" / "продолжить" / "учим X"               │
└────────────────────┬────────────────────────────┘
                     ▼
          ┌──── Тема указана? ────┐
          │                       │
         Нет                     Да
          │                       │
    Смотрит progress.md     Автоопределение:
    + dependency-graph      тема / подтема / блок
    → предлагает тему       → если не найдена:
          │                   спросить или создать
          ▼                       │
    ┌─────────────────────────────┘
    ▼
  Есть learning-plan.md?
    │
   Нет → Калибровка (2-3 вопроса) → генерация плана
   Да  → Читает plan + session.md → находит где остановились
    │
    ▼
  ┌─────────── ЦИКЛ ОБУЧЕНИЯ ──────────┐
  │                                      │
  │  1. Объясняет блок                   │
  │     (аналогия → теория → код)        │
  │     + ★ Insight с нюансами           │
  │                                      │
  │  2. Проверка (2-3 вопроса)           │
  │     ✅ → следующий блок              │
  │     ❌ → наводящие вопросы,          │
  │          пробел → weak-spots.md      │
  │                                      │
  │  3. Практика (если блок требует)     │
  │     задача → проверка решения        │
  │                                      │
  │  4. Микрофидбек:                     │
  │     "Блок понятен? (да/частично/нет)"│
  │     → записать в session.md          │
  │     → обновить learner-profile.md    │
  │                                      │
  │  5. "Дальше?" → ждёт подтверждения   │
  │                                      │
  └──────────┬───────────────────────────┘
             ▼
  Все блоки пройдены?
    │
   Да → Итоговая проверка (5 вопросов)
       → Оценка уровня
       → Обновить все файлы
       → Предложить следующую тему
```

### learning-plan.md (для каждой подтемы)

```markdown

> Блок: 0/N | Последняя сессия: —

<!-- Заполняется при первом входе в тему -->
- Базовые концепции: ⬜

## Блоки

- [ ] **Блок 1: [название]**
  Содержание: ...

  Проверка: — вопросов

- [ ] **Блок 2: [название]**
  ...

- [ ] **Блок N: Практика + итоговая проверка**
```

### session.md (для каждой подтемы)

```markdown
# [Тема] — Сессии

## Сессия 1 (YYYY-MM-DD)

### Блок 1: [название]

**Claude:** [объяснение]
**Адиль:** [вопрос/ответ]
**Claude:** [ответ]

### Проверка Блока 1

**Claude:** Вопрос: ...
**Адиль:** ...
**Claude:** ✅ / ❌ + объяснение

**Фидбек:** понятно ✅

---

## Сессия 2 (YYYY-MM-DD)

### Блок 2: [название] (продолжение)
...
```

#### Углубления в session.md

Когда ученик спрашивает что-то за пределами текущего блока:

```markdown
**Адиль:** А как float хранится в памяти? IEEE 754?

#### ↳ Углубление: IEEE 754
**Claude:** Формат IEEE 754 состоит из...
**Адиль:** Понял, давай дальше

### Блок 2 (продолжение)
```

Learning-plan НЕ меняется при углублениях. Если углубление выявляет пробел → weak-spots.md.

## 4. Автоопределение темы

Когда ученик пишет `учим X`:

1. **Точное совпадение** — X это название подтемы → открыть
2. **Частичное совпадение** — X это блок внутри темы (IReadOnlyList → блок в arrays-lists) → открыть тему на этом блоке
3. **Ключевое слово** — поиск по summary.md, learning-plan.md всех тем
4. **Не найдено** — спросить: «Это новая тема? Куда отнести в дереве?»

При создании новой темы Claude:
- Определяет место в дереве (раздел, уровень)
- Создаёт директорию с файлами
- Генерирует learning-plan.md
- Обновляет roadmap.md и progress.md родительского раздела

---

## 5. Авторазбиение тем

Когда Claude генерирует learning-plan и тема оказывается слишком большой (>7 блоков или явно составная):

1. Claude предлагает: «SQL — слишком широкая тема. Предлагаю разбить на: SELECT & Filtering, JOIN-ы, Агрегация, Оконные функции, CTE, Оптимизация»
2. Ждёт подтверждения
3. Создаёт поддиректории с файлами
4. Создаёт родительский overview.md + progress.md + cheatsheet.md
5. Обновляет roadmap.md

### Иерархия файлов — двухуровневая модель

| Уровень | Файлы | Назначение |
|---------|-------|-----------|
| **Листовой** (joins, select) | Все 9 файлов (summary, session, learning-plan...) | Полноценное обучение |
| **Родительский** (sql) | overview.md, cheatsheet.md, progress.md | Агрегация, навигация |

Родительский cheatsheet — **агрегация** из дочерних (генерируется/обновляется при завершении подтем).

## 6. Обработка переходов между темами

Когда в процессе обучения JOIN-ам ученик спрашивает про SELECT:

| Ситуация | Действие |
|----------|---------|
| Короткий вопрос (1-2 сообщения) | Ответить как углубление, вернуться к текущей теме |
| Полноценный уход (3+ сообщений) | Claude замечает: «Мы ушли в SELECT. Переключиться или вернуться к JOIN?» |
| Переключение подтверждено | Сохранить текущую сессию → открыть/создать сессию по новой теме |

## 8. Startup Protocol

При каждом входе в проект Claude выполняет:

```
1. Прочитать learner-profile.md → кто ученик, стиль обучения
2. Прочитать progress.md трека → что изучено, где остановились
3. Прочитать weak-spots.md → текущие пробелы
4. Найти активную тему (статус 🟡 в progress.md):
   → Если есть: прочитать learning-plan.md + последние 50 строк session.md
   → Если нет: предложить следующую по dependency-graph
5. Поприветствовать:
   "Привет, Адиль. В прошлый раз мы остановились на [тема],
    блок [N]: [название]. Продолжим?"
```

## 9. Инсайты в Learn Mode

Формат `★ Insight` адаптирован под обучение:

```
★ Insight ─────────────────────────────────────
- Связь: [как эта тема связана с уже изученными]
- Нюанс: [что часто путают или упускают]
- Собес: [как это спрашивают на собеседовании]
─────────────────────────────────────────────────
```

Инсайты фокусируются на:
- **Связи между темами** — строить на уже изученном
- **Неочевидные нюансы** — то, что не видно из документации
- **Частые ошибки на собесах** — практическая ценность

## 10. Адаптация под ученика

### Калибровка при входе в тему

Перед генерацией learning-plan Claude задаёт 2-3 вопроса по теме. По ответам:
- Определяет текущий уровень
- Пропускает известные блоки
- Углубляет сложные
- Записывает калибровку в learning-plan.md

### Микрофидбек после каждого блока

Claude спрашивает: «Блок понятен? (да / частично / нет)»

Ответы накапливаются → Claude выявляет паттерны:
- «Адиль лучше усваивает через код»
- «Аналогии работают для абстрактных тем»
- «Предпочитает короткие блоки»

Паттерны записываются в learner-profile.md.

### Связи при объяснении

В learning-plan.md каждый блок содержит поле `Связь`:

```markdown
- [ ] **Блок 2: Generations в GC**
  Связь: ← memory-stack-heap (✅ изучено), → gc-internals
```

Claude строит объяснение на уже изученном: «Помнишь stack vs heap? GC работает только с heap...»

## 11. Скиллы (Claude Code Skills)

Скиллы стандартизируют многошаговые процессы, которые Claude иначе может выполнить неполно.

### learn (`.claude/skills/learn/SKILL.md`)

**Триггеры:** `учим`, `продолжить`, `дальше`, `учим [тема]`

**Что делает:**
1. Startup protocol: загрузить learner-profile, progress, weak-spots
2. Определить тему (автоопределение: точное → частичное → поиск → спросить)
3. Загрузить learning-plan.md + последние записи session.md
4. Если тема новая — калибровка (2-3 вопроса) → генерация learning-plan
5. Войти в цикл обучения (объяснение → проверка → практика → микрофидбек → дальше)
6. При `стоп` / `сохрани` — вызвать save-session

### save-session (`.claude/skills/save-session/SKILL.md`)

**Триггеры:** `сохрани`, `стоп`

**Чеклист (все шаги обязательны):**
1. Дописать диалог в session.md
2. Обновить learning-plan.md (отметить `[x]` пройденные блоки)
3. Обновить summary.md (конспект новых блоков)
4. Обновить progress.md (статус, уверенность)
5. Обновить weak-spots.md (если выявлены пробелы)
6. Обновить learner-profile.md (если есть фидбек/паттерны)
7. Подтвердить: «Прогресс сохранён. Запушить?»

### create-topic (`.claude/skills/create-topic/SKILL.md`)

**Триггеры:** `добавь тему`, `новая тема`, автосоздание при `учим [несуществующая тема]`

**Что делает:**
1. Определить место в дереве (раздел, уровень, родитель)
2. Создать директорию из шаблона `templates/topic/`
3. Сгенерировать learning-plan.md с блоками
4. Обновить roadmap.md
5. Обновить progress.md (добавить строку)
6. Обновить dependency-graph.md (добавить связи)
7. Если есть родитель — обновить родительский progress.md

### rebalance (`.claude/skills/rebalance/SKILL.md`)

**Триггеры:** `разбей тему`, авторазбиение при генерации learning-plan (>7 блоков)

**Что делает:**
1. Проанализировать тему — определить составные части
2. Предложить разбиение на подтемы
3. Ждать подтверждения
4. Создать поддиректории из шаблона `templates/topic/`
5. Создать родительские файлы из шаблона `templates/section/`
6. Перенести существующий контент (session.md, summary.md) в подходящие подтемы
7. Обновить roadmap.md, progress.md, dependency-graph.md

### assess (`.claude/skills/assess/SKILL.md`)

**Триггеры:** `оцени меня`, `оцени меня [тема]`

**Что делает:**
1. Определить тему (или общая оценка)
2. Задать 3-5 вопросов возрастающей сложности
3. По ответам определить уровень (junior/middle/senior/architect)
4. Обновить progress.md (уверенность, уровень)
5. Если пробелы — добавить в weak-spots.md
6. Дать рекомендацию: что изучить для перехода на следующий уровень
